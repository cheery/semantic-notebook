# S-Path-RAG

I used claude to implement a paper describing S-Path-RAG, a form of RAG on knowledge graphs.
The journey was interesting, we had to create a custom database just to handle the freebase dataset with
250GiB worth of entries in it. At first the database took all the memory and crashed but Claude managed
to downscale the amount of memory it requires. Months of effort happened in minutes.

The GLM-5.1 had bootstrapped the project with good code, but it messed up test model's KV-cache so
the whole program produced garbage at first. We found it out and fixed it by prompting claude to
compare the S-RAG implementation with original gemma3 -script.

At the moment, the S-RAG doesn't appear to work like it should. It retrieves irrelevant information
and the model ignores it. Maybe I'll manage to fix it later on. It was an interesting experiment.
