# Semantic Notebook

A terminal-based note organization tool that understands what your notes are about. Write plain markdown files, and Semantic Notebook gives you meaning-based search, automatic topic clustering, and related note suggestions — all running locally on your machine.

## Why use this?

Most note-taking setups give you a folder of files and keyword search. That works until you have fifty notes and can't remember which one discussed that thing about deployment pipelines. You know you wrote about it, but you search "deployment" and the note was titled "CI workflow" and used the word "shipping" instead.

Semantic Notebook solves this by embedding every note with a language model. Search by meaning, not by exact words. Ask for "container orchestration" and it finds your Docker and Kubernetes notes even if they never use that phrase. It also clusters your notes into topics automatically and suggests related notes as you browse — connections you might not have noticed.

Everything stays local. Your notes are plain `.md` files. The embeddings run on your machine. No cloud, no accounts, no lock-in.

## Getting started

**Requirements:** Python 3.11+, pip

**Install dependencies:**

```bash
pip install chromadb sentence-transformers textual rich scikit-learn
```

**Start writing notes:**

Drop `.md` files into the `notes/` directory, or create them with the built-in editor:

```bash
python -m notebook edit my-first-note
```

**Index your notes** (builds embeddings and stores them in ChromaDB):

```bash
python -m notebook index
```

**Launch the TUI:**

```bash
python -m notebook
```

That's it. Browse your notes in the left panel, click or press Enter to view them, press `/` to search, `c` for clusters, `e` to edit, `b` to go back.

## Commands

| Command | What it does |
|---------|-------------|
| `python -m notebook` | Launch the interactive browser |
| `python -m notebook index` | Embed and index all notes |
| `python -m notebook search "query"` | Semantic search from the command line |
| `python -m notebook clusters` | Show auto-detected topic groups |
| `python -m notebook sync [message]` | Update index.md, log.md, and re-index |
| `python -m notebook edit <file>` | Open a note in the built-in editor |
| `python -m notebook build` | Generate a static website |

Every command has `--help` with detailed usage and examples.

## The TUI

Three-column layout: note list | markdown viewer | related notes.

**Key bindings:**

| Key | Action |
|-----|--------|
| `/` | Focus search bar |
| `e` | Edit current note |
| `b` | Go back in history |
| `n` | Create new note |
| `r` | Re-index all notes |
| `c` | Show topic clusters |
| Left/Right | Switch between panels |
| `q` | Quit |

Links in notes, search results, and clusters are all clickable — click or navigate with keyboard and press Enter to follow them.

## The editor

A built-in vim-inspired editor with an always-visible help bar so you never have to memorize keys.

Three modes: **NORMAL** (blue, navigate), **INSERT** (green, type), **COMMAND** (magenta, save/quit). The help bar at the bottom updates to show what keys work in the current mode.

`:w` to save, `:q` to quit, `:wq` for both. Set `$EDITOR` to use something else (vim, nano, etc.) instead.

## How it works

- **Notes** are plain markdown files in `notes/`. The title is the first `# heading`.
- **Embeddings** are generated by [embeddinggemma-300m](https://huggingface.co/google/embeddinggemma-300m) via sentence-transformers, running locally.
- **ChromaDB** stores the vectors persistently in `.notebook/chromadb/`. Indexing is incremental — only changed files are re-embedded.
- **Clustering** uses KMeans on the embeddings, with clusters labeled by TF-IDF keyword extraction.
- **Related notes** are found by querying ChromaDB with a note's own embedding and ranking by cosine similarity.

## Wiki workflow

Semantic Notebook supports a lightweight wiki workflow inspired by [Karpathy's LLM Wiki](https://github.com/karpathy/llm-wiki) pattern. The `notes/CLAUDE.md` schema file tells an LLM assistant how to maintain the notes — adding information, ingesting documents, and keeping cross-links consistent. `index.md` and `log.md` are maintained automatically by the `sync` command.

This means you can point an LLM at a document and say "ingest this into the notebook" and it knows what to do.

## Project structure

```
notebook/
  __main__.py       Entry point (argparse CLI)
  app.py            Textual TUI application
  config.py         Paths, model name, constants
  store.py          Markdown file I/O
  embeddings.py     Embedding model loading and encoding
  index.py          ChromaDB indexing
  search.py         Semantic search
  clusters.py       KMeans clustering with TF-IDF labels
  linker.py         Related note suggestions
  wiki.py           Wiki bookkeeping (index.md, log.md, sync)
  editor/
    app.py          Built-in vim-like text editor
    buffer.py       Text buffer backed by AVL rope
  utils/
    balanced.py     AVL tree and rope data structure
  widgets/          TUI components
notes/              Your markdown notes live here
```
